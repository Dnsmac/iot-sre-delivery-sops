# Kubernetes 底层组件 面试题参考答案

## 一、控制平面

### 1. 核心组件

**控制平面**：apiserver、etcd、scheduler、controller-manager（+ 云上 cloud-controller-manager）。  
**数据平面**：kubelet、kube-proxy、容器运行时（每 Node）。

### 2. apiserver

集群 **唯一 API 入口**；认证/鉴权/准入；Watch 推送；所有状态变更经它写 **etcd**。

### 3. apiserver 挂了

| 能力 | 影响 |
|------|------|
| **已运行 Pod 互访** | **可以**（CNI + kube-proxy 不依赖 apiserver） |
| kubectl / 创建删 Pod / 调度 | **不行** |
| 扩缩容、更新 Deployment | **不行** |

### 4～5. etcd

- 存 **全部** API 对象元数据（`/registry/...`）。  
- **禁止直连**：保证一致性、权限、避免破坏数据；只走 apiserver。

### 6～7. scheduler

- **预选**：过滤不可行节点（资源、污点、亲和、端口…）。  
- **优选**：打分选最优。  
- 失败：资源不足、污点、selector、亲和、拓扑约束等 → `kubectl describe pod` 看 Events。

### 8～9. controller-manager

- 多个 **控制器** 进程集合（Deployment、RS、Node、Endpoint…）。  
- **控制循环**：对比 Spec 与 Status → 调谐。Deployment 控制器创建/维护 RS（D1）。

### 10. Endpoint Controller

根据 Service selector 维护 **Endpoints**（Pod IP 列表），供 kube-proxy 转发（D3）。

### 11. cloud-controller-manager

对接云 API（ELB、云盘、Route）。**裸机**常用 **MetalLB** 代替云 LB，不一定装 CCM。

### 12. 跑在哪

多在 **master/控制平面节点**；业务 Pod 在 **worker**（小集群可能混合）。

## 二、数据平面

### 13. kubelet

本节点 **Pod 生命周期**：拉镜像、启停容器、探针、挂卷、上报状态。

### 14. kubelet vs kube-proxy

| | kubelet | kube-proxy |
|---|---------|------------|
| 管 | Pod/容器 | Service 网络规则 |
| 层 | 运行时 | L4 转发 |

### 15. kube-proxy

监听 Service/Endpoints；**iptables**（规则多时可慢）或 **IPVS**（大规模更好）。详见 D3。

### 16～17. CRI / containerd

**CRI**：kubelet 与运行时之间的标准接口。  
**containerd**：主流运行时；Docker 已不再通过 dockershim 接入，生产推荐 containerd/CRI-O。

### 18. 探针

kubelet 按 PodSpec 探针配置，对容器做 HTTP/TCP/Exec 检查，失败则重启或从 Service 摘掉。

## 三、协作

### 19. apply → Running

```text
kubectl → apiserver → etcd
→ scheduler 绑定 node
→ kubelet 起容器 → 状态上报 apiserver
→ Endpoint Controller → kube-proxy 更新规则
```

### 20. 谁写 etcd / Endpoints

- 对象持久化：**apiserver** 写 etcd。  
- Endpoints：**Endpoint Controller**（在 controller-manager 里）更新。

### 21. iot-web 现网（示例）

apiserver 存 Service/Deploy；controller 调谐 Pod；scheduler 绑定 node；kubelet 起容器；MetalLB 分 VIP（D2）；kube-proxy 转发。

## 四、HA

### 22. 边界

控制平面 **决策与存储**；数据平面 **执行与转发**。业务容器在 Node。

### 23. etcd 奇数

**Raft 多数派**；偶数节点容错不增反浪费。

### 24. apiserver HA

多 apiserver 实例 + **LB/VIP** 对外 6443。

### 25. Node NotReady

Node Controller 标记；一段时间后 **驱逐** 该节点 Pod（若未容忍）。

## 五、陷阱

| # | 简答 |
|---|------|
| F1 | **不会**；scheduler 绑定 node，kubelet 只跑已分配的 |
| F2 | **不会立刻全停**；已运行 Pod 继续跑，但副本补足、Endpoint 更新等停滞 |
| F3 | **不经过**；业务 Pod 间走 CNI/kube-proxy |
| F4 | Controller 在控制平面；**Speaker** 是 DaemonSet 在 **数据平面节点** |
