## W3：40 万设备压测 flagship 故事

**触发**：W2 通过；你负责过压测但面试讲不出数字。  

| 维度 | 内容 |
|------|------|
| **学什么** | 压测指标定义；瓶颈分层（接入/iot-server/MQ/DB/缓存）；调优参数与前后对比 |
| **看什么** | 只读你项目压测报告/笔记；无则翻 Grafana/日志里那段时间（脱敏） |
| **怎么做** | 填表：指标≥3 个数字、瓶颈 1 条、改动 1 条、结果 1 条；画架构简图 |
| **交什么** | `02-EMQX-IoT-Tuning/40万设备压测复盘.md` |
| **简历** | 负责 40W 设备压测，×指标从 A 到 B |

## W4：IoT 协议（只学项目里用过的）

**触发**：W3 通过；协议会用不懂原理。  

| 维度 | 内容 |
|------|------|
| **学什么** | 选 **2 种** 现网协议（如 MQTT+HTTP）；连接、会话、QoS 或等价概念 |
| **看什么** | MQTT 3.1.1：Connect/Subscribe/QoS 章节（30min）；Wireshark 过滤 `mqtt`（实操） |
| **怎么做** | 抓包 1 次；写协议卡片；写「连接打满→限流→降级」链路（对照 iot-server） |
| **交什么** | `协议卡片_*.md`、`连接治理链路.md` |
| **简历** | 物联网连接治理 + 协议名 |
